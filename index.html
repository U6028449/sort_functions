Sorting Algorithms Demonstration

Heap Sort

Heap Sort is a comparison-based sorting technique based on a Binary Heap data structure. It involves building a max heap from the input data and then repeatedly extracting the maximum element from the heap and rebuilding the heap until all elements are sorted.
Advantages: Works well on large database's, and uses memory efficently
Disadvantages: Not stable. More complex to implement compared to simpler algorithms like Bubble Sort.

Radix Sort

Radix Sort is a non-comparative sorting algorithm that sorts numbers by processing individual digits. It processes digits from the least significant to the most significant (LSD Radix Sort) or vice versa (MSD Radix Sort).
Advantages: It is extremley efficient for sorting large numbers of integers.
Disadvantages: Requires additional space for the output array. Not suitable for sorting strings or floating-point numbers without modification.

Counting Sort

Counting Sort is a non-comparative sorting algorithm that works by counting the number of occurrences of each unique element in the input array. The counts are then used to place the elements in the correct position in the output array.
Advantages: It is very stable and requires little modification to run consistently. Efficient for sorting integers with a limited range.
Disadvantages: Requires additional space for the count and output arrays. Not suitable for sorting data with a large range of values or non-integer data.
